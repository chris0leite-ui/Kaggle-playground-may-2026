## Why both Gs failed (the load-bearing lesson)

Both G1 and G2' achieved **low ρ vs PRIMARY** (0.957 / 0.965) — i.e.
the predictions DISAGREE with PRIMARY meaningfully. But min-meta lift
was zero or slightly negative. Disagreement without incremental signal
= **noise**, not orthogonal information.

Root cause: G1 and G2' are both **LGBM bases** — same GBDT class as 13
of the 21 PRIMARY pool bases. Day-12 Option-1 finding showed the GBDT
class is the leakage-eating population (drops −200/−343bp under
GroupKF). Adding new features inside the GBDT class doesn't break the
rank-lock; it just produces another leakage-eater with new noise.

> **Adding new features to LGBM doesn't break rank-lock; switching to a
> different model class does.** Confirms Day-12's strategic implication
> (FM/rule/sparse-LR diversification) and aligns with the parallel
> branch's TabPFN + new-FM-input-features strategy.

## What this empirically rules out

- **γ1 / γ2 / γ3 within-stint relative features in GBDT** — null.
  Stints too short (mean 3.87 laps) for relative features to add
  meaningful signal that LGBM can't already extract from absolute
  features + categorical Stint.
- **γ4 cross-driver intra-race features in GBDT** — null. The +0.29
  block_tyrelife_std row-correlation was real but already implicitly
  captured by the existing pool through Race / Year / LapNumber
  interactions in LGBM trees.
- **β2 stint-grouped LambdaRank** — falsified at smoke. With 63% of
  stint groups all-zero (probe Q1), pairwise loss has signal on only
  37% of groups; net AUC near random.
