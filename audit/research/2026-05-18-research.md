## New candidates (deduped, sorted by EV / cost)

### C1 — Per-(Race) aggregate join from OpenF1 / FastF1

**Source:** S14 (openf1.org/docs), S13 (FastF1).
**Mechanism:** join only **per-Race scalars** to s6e5, not per-Driver.
The 887-Driver synthetic ID will not align to real names (~50 real
drivers across 6 seasons), but the 26 Race levels can. Candidate
scalar features per Race: median green-flag pit-loss seconds, base
rate of VSC/SC incidents, mean track temperature, mean rainfall
indicator. Fold-safe by construction (not target-derived).
**Cost:** ~45 min (data pull + join + 1 LGBM probe).
**Predicted lift:** +0.1 to +0.5 bp (sub-bp because 26 Race levels =
one signal vector and the synth may have already absorbed
LapTime → track-pace dependencies).
**Ledger dedup:** no prior `(Race)`-aggregate external join. The
1.4% match-rate cap was on **Driver**, not Race; this is a different
join key. NEW.

### C2 — DAE with swap-noise on combined train+test

**Source:** Porto Seguro 1st-place (kaggler.com mirror).
**Mechanism:** Jahrer's denoising autoencoder. Swap 15% of features
in each row with values from another randomly-chosen row; the DAE
must reconstruct the original. Then feed hidden activations as
input to small supervised NNs and stack-add as a base. Crucially,
the DAE trains on **train + test rows combined**, which is R25-safe
in our comp (AV-AUC 0.502).
**Cost:** ~2-3 hr Kaggle T4×2.
**Predicted lift:** +0.2 to +0.8 bp (the agent quoted 20-60 bp from
Porto Seguro but our saturation regime caps this).
**Ledger dedup:** d15b is a vanilla DAE on raw inputs (no swap-noise);
ρ=0.948, min-meta +0.79 bp at K=22+1, lived in hedge. **Swap-noise
distinguishes** because the corruption pattern is row-level not
feature-level; this teaches a different representation. NEW variant.

### C3 — Per-(Race, LapNumber) Bayesian shrinkage of PRIMARY OOF/test

**Source:** Domain agent H7; PitStop/PitNextLap decoupling at 19%
(noise floor diagnosis).
**Mechanism:** for each test/OOF row, compute the per-(Race, LapNumber)
mean PRIMARY prediction; shrink the row's prediction toward that
mean with weight w∈[0.1, 0.3]. Pure post-process. Distinct from
EXP-A3-7 (UID smoothing on `Driver_Race_Year`) which **FAILED at
-124 bp** because the UID grouping used the target; H7 groups on
the **prediction** within a Race-LapNumber slice — feature, not
target. Distinct also from "Bayesian group-prior with smoothing"
(noted in HANDOVER) which used PitStop sum as prior; this uses
the PRIMARY prediction itself as prior.
**Cost:** ~5 min (post-process on existing OOF + test).
**Predicted lift:** 0 to +0.3 bp. Likely ties at TIE_ZONE if the
synth's 19% decoupling is i.i.d. across rows.
**Ledger dedup:** A3-7 UID smoothing failed; this is a different
grouping key + different shrinkage target. NEW.

### C4 — UID magic-feature search on base-learner inputs

**Source:** IEEE-CIS Fraud 1st (Deotte/NVIDIA).
**Mechanism:** construct UID = `Driver + '_' + Race + '_' +
floor(LapNumber / W)` for W ∈ {5, 10, 20}; on each fold's training
rows compute 20-40 per-UID groupby aggregates of LapTime, TyreLife,
Position, Cumulative_Degradation (means/stds/max/min/count) — fed
into a single LGBM base learner that joins our K=11 pool. Distinct
from A3-7 UID *smoothing* (that was a post-process on predictions);
this is **UID-aggregates as base-model features**, and IEEE-CIS
quoted +109 bp from a single UID family.
**Cost:** ~25 min CPU (UID construction + aggregates + 1 LGBM 5-fold).
**Predicted lift:** +0.2 to +1.0 bp (saturation-regime discount on
the IEEE-CIS +109 bp). Best EV from this round.
**Ledger dedup:** A3-7 was post-hoc UID-average of *predictions*;
A2-3 was nested-fold TE on 2/3-way concat (different mechanism — TE
output is target-mean, not raw feature aggregates). NEW.
