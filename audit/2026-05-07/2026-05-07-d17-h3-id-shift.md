### What the high AV-AUC actually means

```
train id range: 0..439139   (439140 unique)
test  id range: 439140..627304   (188165 unique)
overlap: 0
```

Train and test ids are **sequentially partitioned with zero overlap**. Any
divisional transform (id // N for N up to ~10000) inherits this trivial
split, so AV-AUC ≈ 1.0 by construction. This is a **labeling convention**
(first 439140 rows are train, next 188165 are test), NOT a
generative-distribution shift between train and test FEATURES. The d12
audit's AV-AUC = 0.502 over actual features confirms train/test feature
distributions are i.i.d.
