# Day-9f — Multi-FM with disjoint feature partitions

> Tier-1 next-step from d9e FFM rejection. d9c FM (k=8 on 8 main-
> effect features, LB 0.95029) is local optimum within unified-FM
> family. Hypothesis: train two FMs on disjoint feature subsets,
> stack both → diversity from feature-partition rather than
> hyperparameter or seed tuning.
> Builder `scripts/d9f_multi_fm.py` (233s wall total).

## Partition design

Two FMs with k=8, 6 epochs (= d9c hyperparams), trained on:

- **FM_A "driver-dynamics"**: D, C, S, T_q5
  (Driver, Compound, Stint, TyreLife-quintile)
- **FM_B "race-context"**: R, Y, Rp_q5, P_q5
  (Race, Year, RaceProgress-quintile, Position-quintile)

Each FM sees 4 active features per row instead of 8. Partition chosen
by domain semantics: A captures driver/strategy/tyre-degradation; B
captures venue/season/race-state.

**Key**: FM_A's ρ vs PRIMARY = **0.487** is the **most-diverse single
base since Day-9 R14 (0.444)**. FM_B's ρ = 0.861 is similar to FM
(0.899) — race-context features overlap with the GBDT pool's
race-aware bases more than driver-dynamics features do.

ρ FM_A vs FM_B = 0.406 — the partitions are *almost orthogonal*,
exactly as designed. The driver-dynamics features and race-context
features predict different aspects of the pit-stop event.

## Why multi-FM works where FFM didn't

FFM (d9e): same 8 features, 4× more parameters (per-field-pair
embeddings). Overfit, 2bp std OOF regression vs FM.

Multi-FM (d9f): same architecture, 2× the embedding tables but each
on a *strict subset of 4 features*. Same total parameter budget per
FM as the single FM (just split across 2 separate FMs).

The signal: the LR meta routes through TWO FM bases at orthogonal
ρ=0.41 to each other, rather than ONE FM base. The orthogonality
comes from domain-grounded feature partitioning, not from random
seed init or k-tuning. **Diversity at the feature-set level, not
the model-instance level.**

## Submission analysis

K=21 swap and K=22 add both predict LB Δ +0.32bp (using d9c's pred-LB
heuristic with ρ in tie band → no offset). Below the +0.5bp slot-
worthy threshold from F1.2's prior, but:

- **d9c FM precedent**: predicted +0.53bp → actual +3.0bp (5.7× upside).
- **d9b L4 precedent**: predicted +0.19bp → actual −0.01bp (TIE).

The d9c precedent is the closest analog — same FM model class, same
LR meta, same OOF→LB amplification regime. If d9f follows the d9c
pattern, expected LB lift is +1.5–3.0bp; if it follows d9b, expected
TIE.

K=21 swap is **structurally cleaner** (smaller pool, FM_d9c removed
because its info is already in FM_A + FM_B). K=22 add keeps the
unified FM as a hedge but the LR demotes it.
